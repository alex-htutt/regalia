# Study materials (problem sets, study guides, practice quizzes)

- **Ground it in the real material** — the lecture slides and any sample exam / Learning Check the user has uploaded — so scope and difficulty match the actual course. Mirror the exam's style and question types.
- **Write original problems.** Don't copy graded or textbook items verbatim. Provide worked solutions at the depth the user asks for: full worked solutions, final answers only, or questions only.
- **Math must render in Obsidian:** inline `$...$`, display `$$...$$`. After writing, verify delimiters are balanced with a quick shell check (count `$`, expect an even number, no stray `$$`).
- **Save + link:** put the note in the fitting subfolder (`problem-sets/`, `exams/`) with proper frontmatter and a `[[wikilink]]` back to the course context file, and add it to that context file's study-materials list.
- **Study-guide structure that works well:** for each concept/test → what it is → when to use it / how to spot the pattern → a short prerequisite quiz → exam-level questions → full solutions. Open with a decision guide ("how to choose") and close with a cumulative "name the technique" set that mixes everything the way the real exam does.
- **Respect the AI policy:** study support and original practice only — never answers meant to be submitted for graded work.
