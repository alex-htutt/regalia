# Practice exams

A practice exam here is a **topic-by-topic study set** in the style of a well-built study guide (e.g. a physics "Exam 2" guide with Theory + Mini Practice Problems + Exam-Level Problems) — each topic scaffolded from its prerequisites up to an exam-level question, with answers shown. Not a bare timed paper.

## 1. Establish the topic list
Get the topics the exam will cover:
- from the user directly, or
- inferred from past exams when enough are available (a floor of ~3) — take the concepts and question types that recur across them. With fewer than that, ask the user rather than guessing a blueprint from one or two.

## 2. For each topic, build a three-tier section
- **Key Concepts** — all prerequisite information needed to solve the topic's problems: definitions, the governing formulas and when each applies, sign conventions, and common pitfalls/tips.
- **Mini Practice Problems** — a few short, single-step questions that drill those prerequisites in isolation, each with a worked answer.
- **Exam-Level Problem** — one harder, multi-step question in the real exam's style, with a hint and a full worked solution.

## 3. Assemble
- Title the document for the exam and list the topics up front, then the per-topic sections in order.
- Save to `exams/` with frontmatter + a `[[wikilink]]` to the course context file, and add it to that context file's study-materials list.

Shared rules (see `study-materials.md`): write original problems (never copy graded/textbook items verbatim), use Obsidian LaTeX with a balanced-`$` check, and respect the course AI policy — practice only, never submittable answers.
